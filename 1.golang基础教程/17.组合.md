### 1. 什么是组合？

Go **没有继承**（inheritance），取而代之的是**组合**（composition）。

组合的方式是：把一个结构体**嵌入**到另一个结构体中，从而「继承」被嵌入结构体的字段和方法。

```go
type B struct{}

func (b B) method2() {}

type A struct {
    B              // 把 B 嵌入 A —— 这就是"组合"
}

func (a A) method1() {}
```

> 📖 **新手友好补充**
>
> 🔍 **生活类比：零件组装**
>
> 想象你在造一辆汽车：
> - 你不需要从头发明发动机，直接把**现成的发动机**装进车身就好
> - 汽车（A）拥有发动机（B），发动机能做的事，汽车也能做
>
> 这就是组合的思路：**A 包含 B，A 就拥有了 B 的所有能力**。
>
> 💡 **为什么 Go 不用继承，而用组合？**
>
> 继承（`class A extends B`）会让 A 和 B **强绑定**，B 的所有行为（包括你不想要的）都强制带给 A，牵一发动全身。
>
> 组合更灵活：你可以挑选嵌入哪些结构体，A 只获得你明确放进去的东西，耦合度更低，也更容易测试。
>
> ⚠️ **新手常见误区**
>
> Go 没有 `extends` 关键字，嵌入不是继承，不要用继承的思维去套组合：
> ```go
> // ❌ Go 没有这种写法
> type A struct extends B {}
>
> // ✅ Go 的写法：直接把类型名放在 struct 里，不写字段名
> type A struct {
>     B
> }
> ```

### 2. 字段和方法的提升 (Promotion)

嵌入后，被嵌入结构体（B）的**字段和方法**会自动「提升」到外层结构体（A），可以直接通过 A 访问，无需写 `.B.xxx`。

```go
type Engine struct {
    Horsepower int
}

func (e Engine) Start() {
    fmt.Println("引擎启动！")
}

type Car struct {
    Engine          // 嵌入 Engine
    Brand string
}

func main() {
    c := Car{
        Engine: Engine{Horsepower: 200},
        Brand:  "Go汽车",
    }

    // 直接访问 Engine 的字段和方法（提升）
    fmt.Println(c.Horsepower) // 等价于 c.Engine.Horsepower
    c.Start()                 // 等价于 c.Engine.Start()
}
```

> 📖 **新手友好补充**
>
> 📝 **最小可运行示例**
>
> ```go
> package main
>
> import "fmt"
>
> type Walker struct{}
>
> func (w Walker) Walk() {
>     fmt.Println("正在走路")
> }
>
> type Person struct {
>     Walker          // 组合进来
>     Name string
> }
>
> func main() {
>     p := Person{Name: "Alice"}
>     p.Walk()               // 直接调用！不需要写 p.Walker.Walk()
>     p.Walker.Walk()        // 这样写也可以，效果一样
> }
> ```
>
> **输出：**
> ```
> 正在走路
> 正在走路
> ```
>
> ⚠️ **新手常见误区**
>
> 「提升」只是语法糖，**底层仍然是 `.B.xxx`**。当命名有歧义时，Go 要求你明确写出路径：
>
> ```go
> type A struct{ Name string }
> type B struct{ Name string }
>
> type C struct {
>     A
>     B
> }
>
> c := C{}
> // fmt.Println(c.Name) // ❌ 编译报错：ambiguous selector c.Name
> fmt.Println(c.A.Name) // ✅ 必须指明是哪个的 Name
> fmt.Println(c.B.Name) // ✅
> ```

### 3. 组合实现接口

组合最强大的地方在于：外层结构体会**自动继承**内层嵌入结构体实现的接口，而无需重复写相同的方法。

```go
type Stringer interface {
    String() string
}

type Base struct{}

func (b Base) String() string {
    return "我是 Base"
}

type Child struct {
    Base           // 嵌入 Base
}

// Child 自动满足 Stringer 接口，因为 Base 已经实现了 String()
```

> 📖 **新手友好补充**
>
> 📝 **完整可运行示例**
>
> ```go
> package main
>
> import "fmt"
>
> // 接口：只要能叫，就是 Animal
> type Animal interface {
>     Sound() string
> }
>
> // Dog 实现了 Animal 接口
> type Dog struct{}
>
> func (d Dog) Sound() string {
>     return "汪汪！"
> }
>
> // GuideDog 是导盲犬，组合了普通狗的能力，再扩展自己的行为
> type GuideDog struct {
>     Dog                    // 嵌入 Dog，自动获得 Sound() 方法
>     Owner string
> }
>
> func (g GuideDog) Guide() {
>     fmt.Println(g.Owner, "的导盲犬正在引路")
> }
>
> func makeSound(a Animal) {
>     fmt.Println(a.Sound())
> }
>
> func main() {
>     gd := GuideDog{Owner: "Alice"}
>
>     gd.Guide()              // GuideDog 自己的方法
>     gd.Sound()              // 提升自 Dog 的方法
>     makeSound(gd)           // GuideDog 自动满足 Animal 接口！
> }
> ```
>
> **输出：**
> ```
> Alice 的导盲犬正在引路
> 汪汪！
> 汪汪！
> ```

### 4. 方法重写（覆盖）

如果外层结构体定义了**与嵌入结构体同名的方法**，外层的方法会**覆盖**内层的方法（但内层方法依然可以通过完整路径访问）。

```go
type Base struct{}

func (b Base) Hello() {
    fmt.Println("Hello from Base")
}

type Child struct {
    Base
}

// Child 重写了 Hello 方法
func (c Child) Hello() {
    fmt.Println("Hello from Child")
}

func main() {
    c := Child{}
    c.Hello()         // 输出：Hello from Child（外层覆盖内层）
    c.Base.Hello()    // 输出：Hello from Base（仍然可以显式访问内层）
}
```

> 📖 **新手友好补充**
>
> 🔍 **类比：公司规章 vs 部门规章**
>
> - 公司（Base）有一套通用的请假流程（`Hello` 方法）
> - 某个部门（Child）有自己的请假流程，**覆盖**了公司通用的
> - 但如果你绕过部门直接走公司流程（`c.Base.Hello()`），公司原来的流程依然存在
>
> ⚠️ **注意**：这不是真正的多态覆盖，只是**名称遮蔽**：
>
> ```go
> // 假设有接口
> type Greeter interface {
>     Hello()
> }
>
> var g Greeter = Child{}
> g.Hello()  // 输出：Hello from Child ✅
>
> var g2 Greeter = Child{}.Base
> g2.Hello() // 输出：Hello from Base ✅
> ```

### 5. 总结

| 概念 | 说明 |
|---|---|
| **嵌入** | 把结构体类型直接写在另一个 struct 里，不带字段名 |
| **提升** | 嵌入类型的字段和方法可以直接通过外层结构体访问 |
| **接口复用** | 外层结构体自动获得嵌入类型实现的接口，无需重写方法 |
| **方法覆盖** | 外层定义同名方法时，外层方法优先；内层仍可通过 `.内层类型.方法名` 访问 |
| **无继承** | Go 没有 `extends`，组合是唯一的代码复用机制，比继承更灵活 |
