Q1: type 结构体名称 struct{

}

> 📖 **新手友好补充**
>
> 🔍 **生活类比**：
> 结构体就像一张**表格模板**。比如一张"用户信息表"，里面有姓名、年龄、邮箱等字段——结构体就是用来把这些相关的数据"打包"成一个整体的。
>
> 💡 **为什么要有结构体？**
> 假设你要描述一个人，没有结构体你得分开存：
> ```go
> name := "Alice"
> age  := 25
> email := "alice@example.com"
> ```
> 三个变量散落各处，很难管理。有了结构体，可以优雅地放在一起：
> ```go
> type User struct {
>     Name  string
>     Age   int
>     Email string
> }
> ```
>
> 💡 **字段名首字母大写 vs 小写**：
> - 首字母**大写**（如 `Name`）：该字段可以被包外部访问（公开）
> - 首字母**小写**（如 `name`）：该字段只能在当前包内访问（私有）
> - 在同一个包内练习时感受不到区别，但写库或大项目时很重要

Q2：
结构体名称{
    name : name,
}

结构体名称.name = "wang"

> 📖 **新手友好补充**
>
> 结构体有两种初始化方式，以及两种赋值方式：
>
> ```go
> type User struct {
>     Name  string
>     Age   int
>     Email string
> }
>
> // 初始化方式1：字面量（指定字段名，推荐，顺序可以任意）
> u1 := User{
>     Name:  "Alice",
>     Age:   25,
>     Email: "alice@example.com",
> }
>
> // 初始化方式2：按字段顺序赋值（不推荐，字段一多很容易写错位置）
> u2 := User{"Bob", 30, "bob@example.com"}
>
> // 初始化后修改字段值：用 . 访问
> u1.Age = 26
> u1.Email = "new@example.com"
>
> fmt.Println(u1) // {Alice 26 new@example.com}
> ```
>
> ⚠️ **新手常见误区**：
> - 方式2（按顺序）在结构体字段增减或调换顺序时，很容易静默地出错（值错位但不报错），所以**强烈推荐始终使用方式1（指定字段名）**。
>
> ---
>
> 💡 **值类型 vs 指针类型**：
> 结构体是**值类型**，赋值或传参时会复制整个结构体（改副本不影响原来的）：
> ```go
> u3 := u1          // 复制了一份
> u3.Name = "Carol"
> fmt.Println(u1.Name) // "Alice" ← 原来的没有变
> fmt.Println(u3.Name) // "Carol"
> ```
> 若想共享同一份数据（或避免大结构体复制的性能开销），用指针：
> ```go
> u4 := &u1         // u4 是指向 u1 的指针
> u4.Name = "Diana" // 等价于 (*u4).Name = "Diana"，Go 自动解引用
> fmt.Println(u1.Name) // "Diana" ← 原来的也变了
> ```
>
> ---
>
> 💡 **给结构体绑定方法（Method）**：
> 结构体不只是数据容器，还可以绑定函数（称为"方法"），让它具有行为：
> ```go
> // 值接收者（方法内修改不影响原结构体）
> func (u User) Greet() string {
>     return "Hi, I'm " + u.Name
> }
>
> // 指针接收者（方法内修改会影响原结构体，常用）
> func (u *User) Birthday() {
>     u.Age++
> }
>
> u := User{Name: "Alice", Age: 25}
> fmt.Println(u.Greet()) // "Hi, I'm Alice"
> u.Birthday()
> fmt.Println(u.Age)     // 26
> ```
> 💡 规则：如果方法需要修改结构体的字段，用**指针接收者** `(u *User)`；只是读数据，用值接收者 `(u User)` 即可。
