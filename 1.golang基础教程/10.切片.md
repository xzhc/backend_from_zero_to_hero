Q1: 区别
数组len是固定； 切片是动态；

> 📖 **新手友好补充**
>
> 🔍 **生活类比**：
> - **数组** 像一排固定座位的电影院，买票时座位数就定死了，多一个人都坐不下。
> - **切片** 像一个可以随时加椅子的会议室，人多了就往里搬椅子，不受数量限制。
>
> 💡 **为什么要有切片？**
> 很多时候我们事先不知道要存多少数据（比如从数据库查出来的记录数），数组的固定长度就很不方便。切片解决了这个问题——让 Go 代码能优雅地处理"数量不确定"的场景。

Q2: 声明
var s []string 

var s = make([]string, len,cap)

> 📖 **新手友好补充**
>
> 切片有三种常见声明方式，各有适用场景：
>
> ```go
> // 方式1：声明一个空切片（nil 切片，长度为 0）
> var s1 []string
>
> // 方式2：用 make 创建，指定初始长度和容量
> // len=3 表示有3个元素（初始值为""），cap=5 表示底层最多能放5个再扩容
> s2 := make([]string, 3, 5)
>
> // 方式3：直接用字面量初始化（最常用）
> s3 := []string{"apple", "banana", "cherry"}
> ```
>
> 🔍 **len 和 cap 是什么？**
> - `len`（length）：切片当前有几个元素，可以用 `len(s)` 获取
> - `cap`（capacity）：底层数组最多能放几个元素，可以用 `cap(s)` 获取
> - 类比：`len` 是抽屉里现在放了几件衣服，`cap` 是抽屉最多能放几件
>
> ⚠️ **新手常见误区**：
> `var s []string` 声明后 s 是 `nil`，但 `var s = []string{}` 声明后 s 是空切片（非 nil）。两者 `len` 都是 0，但 `nil` 判断结果不同：
> ```go
> var a []string     // nil 切片
> b := []string{}    // 空切片
> fmt.Println(a == nil) // true
> fmt.Println(b == nil) // false
> ```

Q3：append
s1 = append(s, "new")

> 📖 **新手友好补充**
>
> 💡 **为什么 append 要把结果赋值回去？**
> Go 的 `append` 不会修改原切片，而是返回一个新切片。所以必须写 `s = append(s, ...)` 才能保留追加后的结果。
>
> ```go
> s := []string{"a", "b"}
> s = append(s, "c")          // 追加单个元素
> s = append(s, "d", "e")     // 追加多个元素
> s2 := []string{"f", "g"}
> s = append(s, s2...)         // 追加另一个切片（注意 ... 展开符）
> fmt.Println(s) // [a b c d e f g]
> ```
>
> ⚠️ **新手常见误区**：
> 追加另一个切片时，必须用 `...` 展开，直接写 `append(s, s2)` 会报类型错误。
>
> 🔍 **扩容机制简介**：
> 当 `len` 超过 `cap` 时，Go 会自动分配一块更大的内存（通常是原来的 2 倍），把旧数据复制过去。这个过程对使用者透明，但要知道**扩容后新切片和旧切片不再共享同一块内存**。

Q4：切片截取（子切片）

```go
s := []int{0, 1, 2, 3, 4}
s1 := s[1:3]  // 取索引 1 到 2（不含 3），结果：[1 2]
s2 := s[:2]   // 从头到索引 1，结果：[0 1]
s3 := s[2:]   // 从索引 2 到末尾，结果：[2 3 4]
```

> 📖 **新手友好补充**
>
> 🔍 **生活类比**：
> 切片截取就像用剪刀剪一段绳子：`s[1:3]` 表示"从第 2 个节点剪到第 3 个节点（左闭右开）"。
>
> ⚠️ **重要陷阱 —— 子切片共享底层数组**：
> 截取得到的子切片和原切片**共用同一块内存**，修改子切片会影响原切片！
> ```go
> s := []int{0, 1, 2, 3, 4}
> s1 := s[1:3]
> s1[0] = 99
> fmt.Println(s) // [0 99 2 3 4]  ← 原切片也被改了！
> ```
> 如果不想影响原切片，用 `copy` 做深拷贝：
> ```go
> s1 := make([]int, 2)
> copy(s1, s[1:3])
> ```
