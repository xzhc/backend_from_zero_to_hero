## Q1: 如何声明函数

关键字：`func`（function 的缩写）

**语法结构：**
```
func 函数名(参数名 参数类型, ...) 返回值类型 {
    // 函数体
}
```

**参数的两种写法：**
```go
// 写法一：每个参数单独写类型
func add(a int, b int) int {
    return a + b
}

// 写法二：相同类型的参数可以合并写（省略前面的类型）
func add(a, b int) int {
    return a + b
}
```

> 📖 **新手友好补充**
>
> 🔍 **生活类比**：函数就像一台"自动售货机"。你往里面投入（参数），它经过内部处理后给你吐出（返回值）结果。你不需要知道机器内部怎么运转，只需要知道"投什么进去、拿什么出来"。
>
> 💡 **为什么要有函数？** 假设你要计算 3 对数字之和，没有函数你要写 3 次加法逻辑；有了函数只需定义一次，调用三次。函数的核心价值是**复用**和**封装**。
>
> 📝 **最小可运行示例：**
> ```go
> func greet(name string) string {
>     return "你好，" + name
> }
> // 调用：greet("小明") → "你好，小明"
> ```
>
> ⚠️ **新手常见误区**：Go 的类型写在变量名**后面**（和 C/Java 相反）。
> ```go
> // ❌ 错误（C/Java 习惯）
> func add(int a, int b) int { ... }
>
> // ✅ 正确（Go 风格）
> func add(a int, b int) int { ... }
> ```

## Q2: 多返回值

Go 函数可以同时返回**多个值**，这是 Go 的特色之一。

```go
// 返回商和余数
func divide(a, b int) (int, int) {
    return a / b, a % b
}

// 调用
quotient, remainder := divide(10, 3)
fmt.Println(quotient, remainder) // 3 1
```

**最常见的用法：返回结果 + 错误**
```go
func parseInt(s string) (int, error) {
    // ...
}

result, err := parseInt("123")
if err != nil {
    // 处理错误
}
```

> 📖 **新手友好补充**
>
> 🔍 **生活类比**：就像你去超市买东西，收银员同时给你"商品"和"收据"，两样东西一起交给你——这就是多返回值。
>
> 💡 **为什么 Go 要有多返回值？** 其他语言（如 Java）通常通过抛出异常来处理错误，Go 则显式地返回一个 `error` 值。这让错误处理更清晰，调用者必须主动去检查，不会被忽略。
>
> ⚠️ **新手常见误区**：如果你不需要某个返回值，用 `_`（下划线）丢弃它，不能直接不接收：
> ```go
> // ❌ 错误：Go 不允许忽略已声明变量
> result := divide(10, 3)  // 编译报错
>
> // ✅ 正确：用 _ 明确表示"我不需要这个值"
> quotient, _ := divide(10, 3)
> ```

## Q3: 命名返回值

可以给返回值起名字，函数内部直接给它赋值，最后 `return` 不需要带值。

```go
func minMax(nums []int) (min, max int) {
    min, max = nums[0], nums[0]
    for _, n := range nums {
        if n < min {
            min = n
        }
        if n > max {
            max = n
        }
    }
    return // 裸 return，自动返回 min 和 max
}
```

> 📖 **新手友好补充**
>
> 💡 **什么时候用命名返回值？** 当函数逻辑复杂、返回值含义不明显时，命名返回值起到"文档注释"的作用，让读代码的人一眼就知道返回的是什么。
>
> ⚠️ **新手常见误区**：命名返回值虽然方便，但"裸 return"在函数较长时会降低可读性（不知道返回了什么）。建议在**短函数**中使用，长函数里还是显式写出返回值更清晰。

## Q4: 可变参数

用 `...类型` 表示可以接收任意数量的该类型参数，函数内部当作切片（slice）使用。

```go
func sum(nums ...int) int {
    total := 0
    for _, n := range nums {
        total += n
    }
    return total
}

sum(1, 2, 3)       // 6
sum(1, 2, 3, 4, 5) // 15

// 也可以把切片展开传入
nums := []int{1, 2, 3}
sum(nums...)  // 注意：切片后面要加 ...
```

> 📖 **新手友好补充**
>
> 🔍 **生活类比**：就像点外卖备注"不要辣、不要葱、不要香菜"——可以写任意多条备注，餐厅统一收下处理。
>
> ⚠️ **新手常见误区**：可变参数必须是函数**最后一个**参数：
> ```go
> // ❌ 错误
> func f(nums ...int, name string) {}
>
> // ✅ 正确
> func f(name string, nums ...int) {}
> ```

## Q5: 匿名函数与闭包

**匿名函数**：没有名字的函数，可以赋值给变量或直接调用。

```go
// 赋值给变量
add := func(a, b int) int {
    return a + b
}
fmt.Println(add(1, 2)) // 3

// 定义后立即调用（IIFE）
result := func(x int) int {
    return x * x
}(5)
fmt.Println(result) // 25
```

**闭包**：匿名函数可以"捕获"并记住它外部的变量。

```go
func counter() func() int {
    count := 0
    return func() int {
        count++   // 捕获并修改外部变量 count
        return count
    }
}

c := counter()
fmt.Println(c()) // 1
fmt.Println(c()) // 2
fmt.Println(c()) // 3
```

**深度解析：`counter` 闭包是如何工作的？**

**① 函数签名：返回值本身是一个函数**

```go
func counter() func() int {
//             ^^^^^^^^^^^
//             返回类型：一个"无参数、返回 int"的函数
```

`counter` 是普通函数，但它的**返回值类型**是 `func() int`，即"不接受参数、返回 int 的函数"。这是 Go 函数作为一等公民（first-class citizen）的体现。

**② 局部变量 `count` 的"逃逸"**

```go
count := 0
```

按常规理解，函数执行完毕后局部变量就会被销毁。但闭包打破了这个规则：Go 的**逃逸分析（escape analysis）**会检测到 `count` 被返回的匿名函数引用，从而将它自动分配到**堆（heap）**上，而非栈上，保证其生命周期跟着闭包走。

**③ 内存模型**

```
调用 counter() 时：
┌─────────────────────────────────┐
│  counter() 的栈帧               │
│  ┌─────────────────────────┐   │
│  │  count = 0  (堆上分配)  │ ←── 被闭包持有，不随栈销毁
│  └────────────┬────────────┘   │
└───────────────┼─────────────────┘
                │ return
                ↓
┌─────────────────────────────┐
│  匿名函数（闭包）            │
│  内部持有指向 count 的引用   │ → count 实际存储在堆上
└─────────────────────────────┘
```

**④ 运行过程追踪**

```go
c := counter()   // 创建闭包，count = 0
c()              // count++ → 1，返回 1
c()              // count++ → 2，返回 2
c()              // count++ → 3，返回 3
```

**⑤ 多个实例彼此独立**

每次调用 `counter()` 都会创建一个**全新的** `count` 绑定，多个实例互不干扰：

```go
c1 := counter()
c2 := counter()
c1() // → 1  （c1 自己的 count）
c1() // → 2  （c1 自己的 count）
c2() // → 1  （c2 独立的 count，从 0 开始）
```

> 一句话总结：**闭包 = 函数 + 它捕获的外部变量。** `counter()` 每次调用都创建一个新的"函数 + count 绑定对"，`count` 跨调用持久存在，形成了**带状态的函数**——这正是闭包最核心的价值。

> 📖 **新手友好补充**
>
> 🔍 **生活类比（闭包）**：闭包就像一个"带记忆的函数"。想象你有一个计步器，每次按下按钮它就在之前的步数上加 1，并记住新的步数——这就是闭包的效果，函数"记住"了上次的状态。
>
> 💡 **为什么要有闭包？** 闭包让你可以创建**有状态的函数**，而不需要定义一个完整的结构体。在需要简单状态管理（如计数器、生成器）时非常方便。
>
> ⚠️ **新手常见误区**：闭包捕获的是变量的**引用**，不是值的拷贝。在循环中创建闭包时要特别小心：
> ```go
> // ❌ 常见陷阱：所有闭包共享同一个 i
> funcs := make([]func(), 3)
> for i := 0; i < 3; i++ {
>     funcs[i] = func() { fmt.Println(i) }
> }
> funcs[0]() // 输出 3，而不是 0！
>
> // ✅ 修复：把 i 作为参数传入，创建一份拷贝
> for i := 0; i < 3; i++ {
>     i := i  // 遮蔽外部 i，创建新变量
>     funcs[i] = func() { fmt.Println(i) }
> }
> funcs[0]() // 输出 0 ✅
> ```
