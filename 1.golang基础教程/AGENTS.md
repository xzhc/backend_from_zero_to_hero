# 项目约束 (Go 基础教程)

## 项目目标
本项目是学习 Go 语言基础知识的教程，目标是通过阅读章节文档、编写测试代码的方式掌握 Go 语言的核心概念。

## 目录结构约定
```
1.golang基础教程/
├── AGENTS.md          # 项目约束文件
├── *.md               # 各章节知识点笔记（如：1.变量.md、2.常量.md）
├── practice/          # 自己动手练习的代码（*_test.go）
│   └── *_test.go
└── experiments/       # 跟随教程实验的示范代码（*_test.go）
    └── *_test.go
```

## AI 行为约束

### 笔记风格
- 所有笔记使用中文编写
- 概念解释要简洁、易懂，适合初学者
- 代码示例要有注释说明
- 每个知识点的 .md 文件放在根目录下

### 代码规范
- 练习代码文件统一放在 `practice/` 目录下，命名为 `<章节号>_<主题>_test.go`
- 示范/实验代码放在 `experiments/` 目录下，命名为 `<章节号>_<主题>_test.go`
- 所有可运行的代码均使用 Go 测试文件格式（`_test.go`），以便通过 `go test` 运行
- 函数命名遵循 Go 惯例（驼峰式命名）

### 学习流程
1. 先阅读概念，生成 .md 笔记
2. 在 `experiments/` 中编写示范代码验证概念
3. 在 `practice/` 中引导用户自己写练习代码，而不是直接给出答案
4. 鼓励用户先自己尝试，再参考示范代码

### 互动风格
- 解释概念时结合 Go 的实际使用场景
- 遇到错误时，先引导用户分析原因，再提供解决方案
- 如用户已有一定理解，可以适当增加难度或补充延伸知识

### 新手友好补充讲解
当用户表示某个概念讲解的 .md 文档对新手不够友好时（例如说"看不懂"、"能不能更易懂一点"、"对新手不友好"、“补充”等），AI 必须：
1. **站在零基础新手的角度**重新审视该文档，找出可能令新手困惑的部分
2. 在原文档中，针对难懂的地方**就地扩展补充**，而不是另起新文件
3. 补充内容须包含以下一项或多项：
   - 🔍 **生活类比**：用日常生活中熟悉的事物来类比抽象概念
   - 💡 **"为什么要这样"**：解释该特性存在的动机或解决的问题
   - 📝 **最小可运行示例**：给出尽可能简短、能直接运行的代码示例
   - ⚠️ **新手常见误区**：列出初学者最容易踩坑的地方并加以说明
4. 补充内容使用 `> 📖 **新手友好补充**` 引用块标注，与原内容视觉上有所区分，方便区分原文与扩展内容
5. 补充后告知用户在文档的哪些位置做了扩展，并询问是否还有不清楚的地方
