内容：
1. context 是网络请求处理的上下文，它可以携带请求的数据；可以实现超时退出、主动退出等操作。

> 📖 **新手友好补充**
>
> ### 🔍 生活类比：外卖订单的"控制中心"
>
> 想象你在美团点了一份外卖：
> - 骑手接单后开始配送（goroutine 启动）
> - 你突然不想要了，点击"取消订单" → 骑手收到通知，停止配送（**主动取消**）
> - 或者超过 30 分钟还没到，系统自动取消 → **超时退出**
> - 订单里还带着你的地址、备注等信息 → **携带值**
>
> **context 就是这个"控制中心"**，它在整个请求链路（主 goroutine → 子 goroutine → 孙 goroutine）中传递"停止信号"和"附加信息"。
>
> ### 💡 为什么需要 context？
>
> 没有 context 时，如果主 goroutine 想停止子 goroutine，需要自己维护 channel 和信号，非常繁琐。
> context 提供了一套**标准化的取消、超时、传值机制**，是 Go 并发编程的惯用写法。
> HTTP 服务器、数据库操作、RPC 调用等场景都大量使用 context。

2. 三种核心用法：取消 / 超时 / 携带值

### 用法一：主动取消（`context.WithCancel`）

```go
ctx, cancel := context.WithCancel(context.Background())
defer cancel() // 调用 cancel() 会向所有子 ctx 发出取消信号
```

> 📖 **新手友好补充**
>
> ### 📝 最小可运行示例：主动取消 goroutine
>
> ```go
> package practice
>
> import (
>     "context"
>     "fmt"
>     "testing"
>     "time"
> )
>
> func TestContextCancel(t *testing.T) {
>     // 创建一个可取消的 ctx，cancel 是"取消按钮"
>     ctx, cancel := context.WithCancel(context.Background())
>
>     go func() {
>         for {
>             select {
>             case <-ctx.Done(): // 监听取消信号
>                 fmt.Println("goroutine 收到取消信号，退出")
>                 return
>             default:
>                 fmt.Println("goroutine 运行中...")
>                 time.Sleep(500 * time.Millisecond)
>             }
>         }
>     }()
>
>     time.Sleep(1500 * time.Millisecond)
>     cancel() // 主动取消，goroutine 会退出
>     time.Sleep(200 * time.Millisecond) // 等待 goroutine 打印退出信息
> }
> ```
> 输出：
> ```
> goroutine 运行中...
> goroutine 运行中...
> goroutine 运行中...
> goroutine 收到取消信号，退出
> ```

### 用法二：超时控制（`context.WithTimeout`）

```go
ctx, cancel := context.WithTimeout(context.Background(), 3*time.Second)
defer cancel() // 即使超时自动触发，也要调用 cancel 释放资源
```

> 📖 **新手友好补充**
>
> ### 📝 最小可运行示例：模拟请求超时
>
> ```go
> func TestContextTimeout(t *testing.T) {
>     // 设置 1 秒超时
>     ctx, cancel := context.WithTimeout(context.Background(), 1*time.Second)
>     defer cancel()
>
>     // 模拟一个耗时 3 秒的操作
>     done := make(chan struct{})
>     go func() {
>         time.Sleep(3 * time.Second) // 模拟慢操作（如数据库查询）
>         close(done)
>     }()
>
>     select {
>     case <-done:
>         fmt.Println("操作完成")
>     case <-ctx.Done(): // 1 秒后触发
>         fmt.Println("超时退出:", ctx.Err()) // context deadline exceeded
>     }
> }
> ```
>
> ### `WithTimeout` vs `WithDeadline` 区别
>
> | 函数 | 参数 | 含义 |
> |---|---|---|
> | `WithTimeout(ctx, 3*time.Second)` | 持续时长 | 从现在起 3 秒后超时 |
> | `WithDeadline(ctx, time.Now().Add(3*time.Second))` | 绝对时间点 | 到达某个具体时刻超时 |
>
> 效果一样，`WithTimeout` 更常用，语义更直观。

### 用法三：携带值（`context.WithValue`）

```go
ctx := context.WithValue(context.Background(), "userID", 42)
val := ctx.Value("userID") // 取值，返回 interface{}，需要类型断言
```

> 📖 **新手友好补充**
>
> ### 📝 最小可运行示例：在调用链中传递请求 ID
>
> ```go
> func TestContextValue(t *testing.T) {
>     // 在请求入口处把 requestID 存入 ctx
>     ctx := context.WithValue(context.Background(), "requestID", "abc-123")
>
>     // 传递给下层函数
>     handleRequest(ctx)
> }
>
> func handleRequest(ctx context.Context) {
>     // 从 ctx 中取出 requestID
>     reqID := ctx.Value("requestID").(string) // 类型断言为 string
>     fmt.Println("处理请求，requestID =", reqID)
> }
> ```
>
> ### ⚠️ 新手常见误区
>
> **误区 1：用 context 传递所有参数（滥用 WithValue）**
> ```go
> // ❌ 不推荐：把普通业务参数塞进 context
> ctx = context.WithValue(ctx, "userName", "Alice")
> ctx = context.WithValue(ctx, "age", 18)
>
> // ✅ context 只应携带请求范围内的元数据，如 requestID、traceID、认证 token
> // 普通参数应该直接通过函数参数传递
> ```
>
> **误区 2：key 用字符串字面量（可能冲突）**
> ```go
> // ❌ 不推荐：不同包都用 "userID" 作为 key，可能冲突
> ctx := context.WithValue(ctx, "userID", 42)
>
> // ✅ 推荐：用自定义类型作为 key，避免跨包冲突
> type contextKey string
> const userIDKey contextKey = "userID"
> ctx := context.WithValue(ctx, userIDKey, 42)
> ```
>
> **误区 3：忘记调用 cancel()**
> ```go
> // ❌ 泄漏资源：没有调用 cancel，ctx 相关资源无法被释放
> ctx, _ := context.WithCancel(context.Background())
>
> // ✅ 始终用 defer cancel()
> ctx, cancel := context.WithCancel(context.Background())
> defer cancel()
> ```
>
> **误区 4：context 应该作为函数的第一个参数传递**
> ```go
> // ✅ Go 惯例：ctx 永远是第一个参数，命名就用 ctx
> func DoSomething(ctx context.Context, userID int) error { ... }
>
> // ❌ 不要把 ctx 放在结构体里存储，也不要作为最后一个参数
> ```

### 🔍 context 的层级关系（父子 ctx）

```
context.Background()          ← 根节点（程序入口处创建）
       │
       ├── WithCancel → ctx1   ← 调用 cancel1() 取消 ctx1 及其所有子孙
       │       │
       │       └── WithTimeout → ctx2  ← ctx1 被取消时，ctx2 也会被取消
       │
       └── WithValue → ctx3    ← 存储键值对
```

> 父 context 取消时，所有子 context 也会自动收到取消信号。
> 子 context 取消时，不影响父 context。
