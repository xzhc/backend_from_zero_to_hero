## 内容

### Q1: 什么是类型？

类型就和 `int`、`bool`、`string` 等一样。比如我们使用 `type` 声明了一个结构体类型：

```go
type Person struct {
    Name string
    Age  int
}
```

那么这个时候，`Person` 就是一个类型。

> 📖 **新手友好补充**
>
> 🔍 **生活类比**：
> 把"类型"想象成**模具**。比如做月饼的模具，定义了月饼的形状（哪里有凸起、哪里凹陷）。
> 有了模具（类型），你就可以用它压出很多块月饼（变量/实例）。
> `int` 是 Go 内置的整数模具，`Person` 是你自己定义的"人"模具。
>
> 💡 **为什么需要自定义类型？**
> Go 内置的 `int`、`string` 等只适合存储简单数据。
> 现实世界中的"人"有姓名、年龄、身高……这些属性需要整合在一起，所以需要自定义类型来组合多个字段。

### Q2: 如何用它来声明一个结构体？

```go
type 结构体名称 struct {
    字段名 字段类型
    // ...
}
```

**完整示例：**

```go
package main

import "fmt"

// 定义一个 Person 类型（结构体）
type Person struct {
    Name string // 姓名
    Age  int    // 年龄
}

func main() {
    // 用 Person 类型创建一个变量（就像用模具压出一个实例）
    p := Person{
        Name: "Alice",
        Age:  25,
    }
    fmt.Println(p.Name, p.Age) // 输出: Alice 25
}
```

> 📖 **新手友好补充**
>
> ⚠️ **新手常见误区：**
> - **误区1**：`type Person struct{...}` 只是定义了"模具"，**并没有创建任何变量**。只有 `p := Person{...}` 这一步才真正创建了数据。
> - **误区2**：结构体字段名**首字母大写**（如 `Name`）才能被其他包访问；小写（如 `name`）只在当前包内可见。（类似"公开"vs"私有"）

### Q3: 如何用它来声明一个接口？

```go
type 接口名称 interface {
    方法名() 返回类型
    // ...
}
```

**完整示例：**

```go
package main

import "fmt"

// 定义一个 Animal 接口
type Animal interface {
    Speak() string // 任何 Animal 类型都必须有 Speak 方法
}

// Dog 类型实现了 Animal 接口
type Dog struct{}

func (d Dog) Speak() string {
    return "汪汪！"
}

// Cat 类型也实现了 Animal 接口
type Cat struct{}

func (c Cat) Speak() string {
    return "喵喵！"
}

func main() {
    animals := []Animal{Dog{}, Cat{}}
    for _, a := range animals {
        fmt.Println(a.Speak())
    }
}
// 输出:
// 汪汪！
// 喵喵！
```

> 📖 **新手友好补充**
>
> 🔍 **生活类比**：
> 接口就像一份**合同/协议**。合同上写着"凡是签了这份合同的，都必须会 `Speak()`"。
> `Dog` 和 `Cat` 都签了 `Animal` 合同，所以它们都必须实现 `Speak()` 方法。
> Go 的接口是**隐式实现**的——不需要显式写 `implements Animal`，只要你的类型拥有接口要求的全部方法，就自动"签了合同"。
>
> 💡 **为什么要有接口？**
> 接口让你写出更通用的代码。上面的例子中，`main` 函数不用关心是 `Dog` 还是 `Cat`，只要是 `Animal` 就能调用 `Speak()`。
>
> ⚠️ **新手常见误区：**
> - 接口里只写**方法签名**（方法名+参数+返回值），**不写具体实现**。
> - Go 接口是**隐式**满足的，不像 Java/C# 需要写 `implements`。

### Q4: `type` 的其他用法——基于已有类型创建新类型

除了结构体和接口，`type` 还可以基于已有类型创建一个**全新的命名类型**：

```go
type Celsius float64    // 摄氏度（基于 float64）
type Fahrenheit float64 // 华氏度（基于 float64）

type MyString string // 基于 string 的新类型
```

> 📖 **新手友好补充**
>
> 💡 **为什么要这样做？**
> 虽然 `Celsius` 和 `Fahrenheit` 底层都是 `float64`，但它们是**不同的类型**，不能直接混用，Go 编译器会阻止你把摄氏度直接当作华氏度传进函数，从而避免低级的单位混淆错误。
>
> ```go
> var c Celsius = 100.0
> var f Fahrenheit = 212.0
> // f = c  // ❌ 编译错误：类型不匹配！
> f = Fahrenheit(c) // ✅ 需要显式类型转换
> ```
>
> 📝 **最小可运行示例：**
> ```go
> package main
>
> import "fmt"
>
> type Celsius float64
>
> func main() {
>     var temp Celsius = 36.6
>     fmt.Printf("体温: %.1f°C\n", temp) // 输出: 体温: 36.6°C
> }
> ```

---

### 小结

| 用法 | 语法 | 作用 |
|:---|:---|:---|
| 声明结构体 | `type Name struct{ ... }` | 组合多个字段，表示复杂数据 |
| 声明接口 | `type Name interface{ ... }` | 定义一组方法约束（合同） |
| 基于现有类型 | `type Name baseType` | 创建语义更清晰的新类型 |
