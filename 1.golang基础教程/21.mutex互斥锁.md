内容：
1. 独占锁，也就是说同一时刻被加锁的对象，只能被一个协程进行操作

> 📖 **新手友好补充**
>
> ### 🔍 生活类比：公共厕所的锁
>
> 想象一个只有一个坑位的公共厕所：
> - 里面有人时，门锁上了 → 其他人只能在外面等
> - 里面的人出来后，门锁开了 → 下一个人才能进去
>
> **互斥锁（Mutex）就是这把锁**。当多个 goroutine 同时想操作同一块数据时，互斥锁保证"同一时刻只有一个 goroutine 能进入"，从而避免数据混乱。
>
> ### 💡 为什么需要互斥锁？
>
> 多个 goroutine **并发**地读写同一个变量时，会出现**竞态条件（Race Condition）**，导致结果不可预知。
>
> 下面的例子展示了**不加锁**时的问题：
> ```go
> // 不安全的并发写操作（不要这样写！）
> var counter int
> for i := 0; i < 1000; i++ {
>     go func() {
>         counter++ // 多个 goroutine 同时修改，结果会出错！
>     }()
> }
> // 最终 counter 的值很可能不是 1000
> ```
> 多个 goroutine 同时执行 `counter++`，这个操作其实包含三步（读→加→写），中途可能被其他 goroutine 打断，导致某些加法操作"丢失"。

---

2. 用法
```go
var mu sync.Mutex   // 注意：Mutex 首字母大写（M 大写，不是 mutex）
mu.Lock()           // 加锁：当前 goroutine 独占，其他 goroutine 在此阻塞等待
defer mu.Unlock()   // 解锁：defer 确保函数结束时一定会释放锁
```

> 📖 **新手友好补充**
>
> ### 📝 最小可运行示例：用互斥锁保护计数器
>
> ```go
> package practice
>
> import (
>     "sync"
>     "testing"
> )
>
> func TestMutex(t *testing.T) {
>     var mu sync.Mutex
>     var counter int
>     var wg sync.WaitGroup
>
>     for i := 0; i < 1000; i++ {
>         wg.Add(1)
>         go func() {
>             defer wg.Done()
>             mu.Lock()           // 加锁：只允许一个 goroutine 进入
>             counter++           // 安全操作
>             mu.Unlock()         // 解锁：让下一个 goroutine 可以进入
>         }()
>     }
>
>     wg.Wait()
>     t.Log("counter =", counter) // 结果稳定为 1000
> }
> ```
>
> ### ⚠️ 新手常见误区
>
> **误区 1：`sync.mutex`（小写）**
> ```go
> // ❌ 错误：mutex 首字母必须大写
> var mu sync.mutex
>
> // ✅ 正确
> var mu sync.Mutex
> ```
>
> **误区 2：忘记 Unlock，导致死锁**
> ```go
> // ❌ 危险：如果中途 return 或 panic，锁永远不会释放
> mu.Lock()
> // ... 一些操作 ...
> mu.Unlock() // 如果上面的代码 panic 了，这行永远不会执行
>
> // ✅ 推荐：用 defer 确保一定解锁
> mu.Lock()
> defer mu.Unlock() // 无论函数如何退出，都会执行解锁
> ```
>
> **误区 3：对同一把锁重复加锁（死锁）**
> ```go
> // ❌ 错误：同一个 goroutine 对同一把锁 Lock 两次，会永远卡住
> mu.Lock()
> mu.Lock() // 死锁！第一把锁还没释放，第二次 Lock 会一直等待
> ```
>
> **误区 4：值拷贝 Mutex**
> ```go
> // ❌ 错误：Mutex 不能被复制，复制后的锁是独立的，失去了互斥效果
> mu1 := sync.Mutex{}
> mu2 := mu1 // 复制了一个新的 Mutex，和 mu1 没有关联！
>
> // ✅ 正确：通过指针传递
> func doSomething(mu *sync.Mutex) { ... }
> ```
>
> ### 🔍 Lock 与 Unlock 的执行流程
>
> ```
> goroutine A         goroutine B
>    │                    │
>    ├─ mu.Lock() ✅      │
>    │  (拿到锁)          ├─ mu.Lock() ⏳
>    │                    │  (阻塞等待 A 释放)
>    ├─ counter++         │
>    │                    │
>    ├─ mu.Unlock()       │
>    │  (释放锁)          ├─ mu.Lock() ✅
>    │                    │  (拿到锁，继续执行)
>    │                    ├─ counter++
>    │                    ├─ mu.Unlock()
> ```
