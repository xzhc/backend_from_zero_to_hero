Q1: (key,value)

> 📖 **新手友好补充**
>
> 🔍 **生活类比**：
> map 就像一本**通讯录**：每个人的名字（key）对应一个电话号码（value）。你通过名字查号码，而不是通过"第几个"来找。
>
> 💡 **为什么要有 map？**
> 数组和切片只能通过下标（0、1、2…）访问数据，而 map 可以用任意有意义的"名字"来查找数据，更符合实际业务需求。例如：记录每个用户的积分、统计每个单词出现的次数。

Q2： make(map[key]value)
举例： map := make(map[string]int)

> 📖 **新手友好补充**
>
> map 有两种常见初始化方式：
>
> ```go
> // 方式1：用 make 创建（推荐，先声明再填数据）
> scores := make(map[string]int)
>
> // 方式2：用字面量直接初始化（适合初始值已知时）
> scores := map[string]int{
>     "Alice": 95,
>     "Bob":   87,
> }
> ```
>
> ⚠️ **新手常见误区**：map **必须初始化后才能使用**！如果只写 `var m map[string]int` 而不初始化，直接写入数据会触发 panic（程序崩溃）：
> ```go
> var m map[string]int
> m["Alice"] = 95 // ❌ panic: assignment to entry in nil map
>
> m := make(map[string]int)
> m["Alice"] = 95 // ✅ 正确
> ```

Q3：map["key1"] = 2

> 📖 **新手友好补充**
>
> ```go
> scores := make(map[string]int)
> scores["Alice"] = 95  // 新增 key
> scores["Alice"] = 100 // 再次赋值会直接覆盖，不会报错
> ```
>
> 💡 map 中同一个 key 只会存在一份，重复赋值等于更新。

Q4：var value = map["key1"]

> 📖 **新手友好补充**
>
> 查询时有两种写法，**强烈推荐第二种**（双返回值）：
>
> ```go
> scores := map[string]int{"Alice": 95}
>
> // 写法1：直接取值（如果 key 不存在，返回该类型的零值，int 是 0）
> v := scores["Bob"]
> fmt.Println(v) // 0 ← 但我们不知道是真的存了 0 还是 key 根本不存在！
>
> // 写法2：双返回值（推荐）
> v, ok := scores["Bob"]
> if ok {
>     fmt.Println("找到了:", v)
> } else {
>     fmt.Println("key 不存在") // 会走这里
> }
> ```
>
> ⚠️ **新手常见误区**：直接用写法1查一个不存在的 key，得到的是零值（0 / "" / false），很容易和真实存储的零值混淆。养成用 `v, ok := map[key]` 的双返回值习惯。

Q5：delete(map,"key1")

> 📖 **新手友好补充**
>
> ```go
> scores := map[string]int{"Alice": 95, "Bob": 87}
> delete(scores, "Alice")
> fmt.Println(scores) // map[Bob:87]
> ```
>
> 💡 删除一个**不存在的 key** 不会报错，Go 会安静地忽略这个操作，无需担心。

---

Q6: clear(map)

> 📖 **新手友好补充**
>
> ```go
> scores := map[string]int{"Alice": 95, "Bob": 87}
> clear(scores)
> fmt.Println(scores) // map[]
> fmt.Println(len(scores)) // 0
> ```
>
> 💡 `clear` 是 Go 1.21 新增的内置函数，比循环逐个删除效率更高，也更简洁。如果你的 Go 版本低于 1.21，可以用重新 make 代替：
> ```go
> scores = make(map[string]int) // 等效于清空
> ```
