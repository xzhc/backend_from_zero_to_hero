### 1. 什么是 Goroutine？

Goroutine 是 Go 语言提供的**轻量级线程**，由 Go 运行时（runtime）管理，而不是操作系统。

你可以同时启动**成千上万**个 goroutine，而不会像系统线程那样消耗大量内存。

> 📖 **新手友好补充**
>
> 🔍 **生活类比：餐厅里的厨师**
>
> 想象一家餐厅：
> - **单线程（顺序执行）**：只有一个厨师，点了牛排就得等牛排做完，才能开始做沙拉，客人等很久
> - **多 Goroutine（并发执行）**：厨师同时接多个任务——牛排在烤箱里烤着，他顺手就把沙拉做了，效率大幅提升
>
> Goroutine 就像是 Go 给你的「同时干多件事」的能力，而且启动成本极低。
>
> 💡 **Goroutine vs 系统线程**
>
> | | 系统线程 | Goroutine |
> |---|---|---|
> | 初始栈大小 | ~1 MB | ~2 KB |
> | 创建成本 | 高（内核调用） | 极低（用户态） |
> | 同时存在数量 | 通常几千 | 可以几十万 |
> | 调度者 | 操作系统 | Go 运行时 |
>
> 这就是为什么 Go 特别适合写**高并发**服务（如 Web 服务器）。

### 2. 如何创建 Goroutine？

在函数调用前加上 `go` 关键字，就会在一个新的 goroutine 中并发执行该函数。

```go
go func()
```

完整语法：

```go
// 方式一：直接 go 一个已定义的函数
go 函数名(参数)

// 方式二：go 一个匿名函数（最常见）
go func() {
    // 并发执行的代码
}()
```

> 📖 **新手友好补充**
>
> 📝 **最小可运行示例**
>
> ```go
> package main
>
> import (
>     "fmt"
>     "time"
> )
>
> func say(msg string) {
>     fmt.Println(msg)
> }
>
> func main() {
>     go say("我在新 goroutine 里执行")  // 并发启动
>     say("我在主 goroutine 里执行")     // 主 goroutine 继续
>
>     time.Sleep(time.Millisecond * 100) // 等一下，让子 goroutine 有机会执行
> }
> ```
>
> **可能的输出（顺序不固定）：**
> ```
> 我在主 goroutine 里执行
> 我在新 goroutine 里执行
> ```
>
> 🔍 **为什么主 goroutine 的那行总是先打印？**
>
> `go say(...)` 只是**发出调度指令**，Go 运行时在合适的时机才会真正切换过去执行。
> 而 `say(...)` 没有 `go`，是同步调用，会立刻在当前 goroutine 里执行完毕。
> 所以主 goroutine 的那行几乎总会先完成，然后 Go 运行时才有机会调度子 goroutine。
>
> 💡 **`time.Sleep` 的局限性**
>
> | 维度 | 说明 |
> |---|---|
> | 作用 | 让主 goroutine 暂停，给子 goroutine 被调度执行的机会 |
> | 缺点 | 靠"猜"一个等待时间，无法保证子 goroutine 一定执行完 |
> | 适用场景 | 学习演示、临时调试 |
> | 生产代码 | 不推荐，应使用 `sync.WaitGroup` |
>
> ⚠️ **新手常见误区：主 goroutine 退出，所有子 goroutine 立即终止**
>
> ```go
> func main() {
>     go fmt.Println("子 goroutine")   // 启动了，但还没来得及执行
>     // main 函数直接结束，程序退出
>     // 子 goroutine 可能根本没有输出！
> }
> ```
>
> **解决方法**：用 `time.Sleep` 临时等待（不推荐用于生产），或用 `sync.WaitGroup`（推荐，见下节）。

### 3. 等待 Goroutine 完成：sync.WaitGroup

`sync.WaitGroup` 是 Go 中用来**等待一组 goroutine 全部完成**的标准工具。

```go
import "sync"

var wg sync.WaitGroup

wg.Add(1)      // 告诉 WaitGroup：有 1 个 goroutine 要等
go func() {
    defer wg.Done()  // goroutine 结束时，通知 WaitGroup：我完成了
    // ... 实际工作 ...
}()

wg.Wait()      // 主 goroutine 阻塞在这里，直到计数归零
```

> 📖 **新手友好补充**
>
> 🔍 **类比：签到板**
>
> 想象老师用签到板管理学生：
> - `wg.Add(N)` → 签到板上记录「有 N 个学生要交作业」
> - `wg.Done()` → 每个学生交完作业，在签到板上划掉一个
> - `wg.Wait()` → 老师站在门口等待，直到签到板上全部划完，才能下班
>
> 📝 **完整可运行示例**
>
> ```go
> package main
>
> import (
>     "fmt"
>     "sync"
> )
>
> func worker(id int, wg *sync.WaitGroup) {
>     defer wg.Done()                        // 函数结束时自动通知完成
>     fmt.Printf("Worker %d 开始工作\n", id)
>     // 模拟一些工作...
>     fmt.Printf("Worker %d 完成工作\n", id)
> }
>
> func main() {
>     var wg sync.WaitGroup
>
>     for i := 1; i <= 3; i++ {
>         wg.Add(1)                          // 每启动一个 goroutine，计数 +1
>         go worker(i, &wg)                  // 注意传指针 &wg
>     }
>
>     wg.Wait()                              // 等待所有 worker 完成
>     fmt.Println("所有 Worker 都完成了！")
> }
> ```
>
> **可能的输出（顺序不固定）：**
> ```
> Worker 1 开始工作
> Worker 3 开始工作
> Worker 2 开始工作
> Worker 2 完成工作
> Worker 1 完成工作
> Worker 3 完成工作
> 所有 Worker 都完成了！
> ```
>
> 🔍 **实际运行结果分析（来自练习代码 `go test -run TestGoroutine`）：**
>
> ```
> Worker 2 开始工作
> Worker 2 完成工作
> Worker 1 开始工作
> Worker 1 完成工作
> Worker 0 开始工作
> Worker 0 完成工作
> 所有 worker 都完成了
> ```
>
> Worker 是按 **2→1→0 倒序**完成的，这说明调度器倾向于优先调度**最近启动**的 goroutine。
> 但这只是当时的调度结果，**不能依赖这个顺序**——每次运行都可能不同。
> 唯一可以保证的是：`所有 worker 都完成了` 一定是**最后一行**，这正是 `wg.Wait()` 的意义所在。
>
> ⚠️ **新手常见误区**
>
> 1. **忘记 `wg.Done()`**：程序会永远阻塞在 `wg.Wait()` 上（死锁）
>    - 推荐用 `defer wg.Done()` 放在 goroutine 函数的第一行，确保一定会执行
>
> 2. **传值而不是传指针**：`sync.WaitGroup` 必须传指针，否则子 goroutine 修改的是副本，`Wait()` 永远不会结束
>    ```go
>    // ❌ 错误：传值
>    go worker(i, wg)
>
>    // ✅ 正确：传指针
>    go worker(i, &wg)
>    ```
>
> 3. **`wg.Add` 放在 goroutine 内部**：如果 `Add` 放进去，主 goroutine 可能在 `Add` 执行前就到达 `Wait()`
>    ```go
>    // ❌ 错误写法
>    go func() {
>        wg.Add(1)   // 可能来不及执行
>        defer wg.Done()
>    }()
>    wg.Wait()
>
>    // ✅ 正确：在启动 goroutine 之前调用 Add
>    wg.Add(1)
>    go func() {
>        defer wg.Done()
>    }()
>    wg.Wait()
>    ```

### 4. 总结

| 概念 | 说明 |
|---|---|
| `go 函数()` | 启动一个新 goroutine 并发执行 |
| goroutine 轻量 | 初始只占 ~2KB，可以启动几十万个 |
| 主 goroutine 退出 = 程序结束 | 子 goroutine 会被强制终止 |
| `sync.WaitGroup` | 等待一组 goroutine 全部完成的标准方式 |
| `wg.Add(n)` | 启动前登记 n 个待等待的 goroutine |
| `wg.Done()` | goroutine 结束时调用，计数 -1，推荐用 `defer` |
| `wg.Wait()` | 阻塞当前 goroutine，直到计数归零 |
