## 内容：

### 1. 函数签名集合

接口是一组**方法签名的集合**（只声明方法名、参数、返回值，不包含具体实现）。

> 📖 **新手友好补充**
>
> 🔍 **生活类比：USB 接口**
>
> 想象 USB 接口的规范：
> - USB 规范（接口）只规定了"插孔的形状、电压、数据协议"——也就是**要满足什么条件**
> - 鼠标、键盘、U盘（具体类型）只要**符合这个规范**，就能插进去用
> - USB 规范本身不关心你是鼠标还是键盘，它只关心你**有没有实现那些约定**
>
> Go 的接口也是一样：接口只规定"你必须有哪些方法"，任何类型只要具备这些方法，就自动实现了这个接口。
>
> 💡 **为什么需要接口？**
>
> 没有接口时，代码和具体类型紧耦合——比如你写了一个函数专门处理"狗叫"，就没法复用来处理"猫叫"。
> 有了接口，你可以写一个处理"任何会叫的动物"的函数，让代码更灵活、可扩展。

### 2. 接口的声明

```go
type 接口名称 interface {
    方法名(参数) 返回值
    // 可以有多个方法
}
```

> 📖 **新手友好补充**
>
> 📝 **最小可运行示例**
>
> ```go
> // 声明一个接口：只要能"叫"，就是 Animal
> type Animal interface {
>     Sound() string
> }
> ```
>
> ⚠️ **新手常见误区**
>
> - 接口里**只写方法签名，不写方法体**（没有花括号 `{}`，没有函数逻辑）
> - 接口**不能有字段**（字段是 struct 的事，interface 只管行为/方法）
>
> ```go
> // ❌ 错误：接口里不能有字段
> type Animal interface {
>     Name string   // 编译报错！
>     Sound() string
> }
>
> // ✅ 正确：接口只有方法签名
> type Animal interface {
>     Sound() string
> }
> ```

### 3. 接口如何被实现

任何类型都可以实现接口，**只要该类型实现了接口中的所有方法，就认为该接口被该类型实现**。

Go 的接口实现是**隐式的**，不需要写 `implements` 关键字。

> 📖 **新手友好补充**
>
> 💡 **"隐式实现"是 Go 的特色**
>
> 和 Java、C# 不同，Go 不需要显式声明"我实现了某个接口"。
> 只要你的类型有接口要求的所有方法，Go 编译器自动认为你实现了它。
>
> 📝 **完整可运行示例**
>
> ```go
> package practice
>
> import (
>     "fmt"
>     "testing"
> )
>
> // 1. 声明接口
> type Animal interface {
>     Sound() string
> }
>
> // 2. 定义两个结构体
> type Dog struct{}
> type Cat struct{}
>
> // 3. 为 Dog 实现接口方法（不需要写 implements！）
> func (d Dog) Sound() string {
>     return "汪汪！"
> }
>
> // 4. 为 Cat 实现接口方法
> func (c Cat) Sound() string {
>     return "喵喵！"
> }
>
> // 5. 接口的威力：同一个函数可以处理所有实现了 Animal 的类型
> func makeSound(a Animal) {
>     fmt.Println(a.Sound())
> }
>
> func TestInterface(t *testing.T) {
>     dog := Dog{}
>     cat := Cat{}
>
>     makeSound(dog) // 输出：汪汪！
>     makeSound(cat) // 输出：喵喵！
> }
> ```
>
> ⚠️ **新手常见误区**
>
> 1. **方法签名必须完全一致**（方法名、参数类型、返回值类型都要匹配）：
>    ```go
>    type Animal interface {
>        Sound() string
>    }
>
>    type Bird struct{}
>    // ❌ 返回值类型不对，不算实现了 Animal
>    func (b Bird) Sound() int { return 1 }
>
>    // ✅ 这才算实现
>    // func (b Bird) Sound() string { return "啾啾！" }
>    ```
>
> 2. **必须实现接口的全部方法**，缺一个都不行：
>    ```go
>    type Mover interface {
>        Walk()
>        Run()
>    }
>    type Snail struct{}
>    func (s Snail) Walk() {} // 只实现了 Walk，没有 Run
>    // ❌ Snail 没有实现 Mover，把 Snail 赋值给 Mover 类型会编译报错
>    ```
>
> 3. **值接收者 vs 指针接收者**（进阶）：
>    - 用**值接收者**实现的方法，值类型和指针类型都满足接口
>    - 用**指针接收者**实现的方法，只有**指针类型**满足接口
>    ```go
>    type Speaker interface { Speak() }
>    type Person struct{}
>
>    func (p *Person) Speak() { fmt.Println("hello") } // 指针接收者
>
>    var s Speaker = &Person{}  // ✅ 正确
>    // var s Speaker = Person{} // ❌ 编译报错
>    ```
