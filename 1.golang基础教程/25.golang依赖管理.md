Q1: 什么是 Go Module？

Go Module 是 Go 语言官方的**依赖管理系统**（Go 1.11 引入，1.16 后成为默认方式）。它解决的核心问题是：当你的项目需要用到别人写的库时，如何管理这些库的版本、下载和更新。

> 📖 **新手友好补充**
>
> 🔍 **生活类比**：
> 想象你在做一道菜（你的项目），需要用到几种调料（第三方库）。
> - 没有依赖管理时：你只知道"需要酱油"，但不知道用哪个牌子、多少克，每次做菜结果可能不一样
> - 有了 Go Module：你有一份**精确的食谱**（`go.mod`），写明"需要 XX 牌酱油，300ml"，任何人按这份食谱做菜，结果都一样
>
> 💡 **为什么需要依赖管理？**
> - **版本锁定**：确保团队每个人用同一版本的库，避免"在我机器上能跑"的问题
> - **依赖传递**：你用的库 A 可能还依赖库 B，Go Module 自动帮你管理这些间接依赖
> - **可重复构建**：任何时间、任何机器，只要有 `go.mod` 和 `go.sum`，就能构建出完全相同的程序

## 核心文件

| 文件 | 作用 |
|------|------|
| `go.mod` | 声明模块名称、Go 版本、直接依赖及其版本 |
| `go.sum` | 记录所有依赖（包括间接依赖）的精确哈希值，用于校验完整性 |

> 📖 **新手友好补充**
>
> 🔍 **`go.mod` vs `go.sum` 的类比**：
> - `go.mod` = **购物清单**：我需要买哪些东西、买哪个版本
> - `go.sum` = **验货单**：每件商品的序列号，收货时用来验证是否买到了正品（防篡改）
>
> ⚠️ **新手常见误区**：
> - `go.sum` 是**自动生成**的，不要手动编辑它
> - 两个文件都应该提交到 Git，这样团队成员能用同一份依赖

Q2: 常用 go mod 命令

## 初始化模块

```bash
# 在项目根目录执行，创建 go.mod 文件
go mod init <模块路径>

# 示例：
go mod init github.com/yourname/myproject
```

> 📖 **新手友好补充**
>
> 💡 **模块路径（module path）怎么取名？**
> - 如果你的代码会发布到 GitHub，用 `github.com/用户名/仓库名`
> - 如果只是本地练习项目，随便取，比如 `example.com/hello` 或 `myproject`
> - 模块路径最终会成为别人 `import` 你代码时的包路径前缀
>
> 📝 **最小可运行示例**：
> ```bash
> mkdir myproject && cd myproject
> go mod init example.com/myproject
> # 生成的 go.mod 内容：
> # module example.com/myproject
> # go 1.21
> ```

## 添加依赖

```bash
# 方式1：直接在代码里 import，然后运行 go mod tidy 自动添加
go mod tidy

# 方式2：手动指定版本下载
go get github.com/some/package@v1.2.3

# 下载最新版本
go get github.com/some/package@latest
```

> 📖 **新手友好补充**
>
> 🔍 **`go get` 和 `go mod tidy` 的区别**：
> - `go get 包名@版本`：**精确**添加一个包，适合想指定版本时
> - `go mod tidy`：扫描你的代码，**自动**添加缺失的依赖、删除未使用的依赖，适合整理依赖
>
> 💡 **推荐工作流**：先在代码里写好 `import`，然后运行 `go mod tidy`，让 Go 自动解决依赖，比手动 `go get` 更方便。
>
> ⚠️ **新手常见误区**：`go get` 在 Go 1.17+ 的语义有变化，现在主要用于**更新依赖版本**，不再用于安装工具。安装工具（如 linter）用 `go install`。

## 版本管理命令

```bash
# 查看所有依赖（包括间接依赖）及其版本
go list -m all

# 查看某个包的所有可用版本
go list -m -versions github.com/some/package

# 升级到最新版本
go get github.com/some/package@latest

# 降级到指定版本
go get github.com/some/package@v1.3.1

# 删除未使用的依赖，补全缺少的依赖
go mod tidy
```

## 查看依赖

```bash
# 列出所有直接和间接依赖
go list -m all

# 以图形方式查看依赖关系
go mod graph

# 下载所有依赖到本地缓存（ci/cd 常用）
go mod download
```

Q3: go.mod 文件解读

```
module example.com/myproject   ← 模块路径（包的导入前缀）

go 1.21                        ← 使用的 Go 版本

require (
    github.com/gin-gonic/gin v1.9.1          ← 直接依赖
    golang.org/x/text v0.14.0 // indirect    ← 间接依赖（由 gin 引入）
)
```

> 📖 **新手友好补充**
>
> 💡 **`// indirect` 是什么意思？**
> 表示这个包不是你的代码直接 `import` 的，而是你的某个依赖包在内部用到的。Go 会把它记录在 `go.mod` 里以锁定版本，但你不需要关心它，`go mod tidy` 会自动维护。
>
> ⚠️ **新手常见误区**：看到 `indirect` 不要删掉它，这是正常现象。

Q4: 语义化版本（Semantic Versioning）

Go Module 使用 **Semver（语义化版本号）**，格式为 `vMAJOR.MINOR.PATCH`：

| 类型 | 含义 | 示例 |
|------|------|------|
| MAJOR（主版本） | 有**不兼容**的 API 变更 | v1 → v2 |
| MINOR（次版本） | 新增功能，**向后兼容** | v1.2 → v1.3 |
| PATCH（补丁版本） | Bug 修复，**向后兼容** | v1.2.3 → v1.2.4 |

> 📖 **新手友好补充**
>
> 🔍 **类比**：
> - MAJOR 升级 = 重新装修房子，原来的家具可能放不进去了
> - MINOR 升级 = 添置了新家具，不影响原有家具
> - PATCH 升级 = 修了一个漏水的水龙头，其他都没变
>
> 💡 **特殊：主版本 v2+ 的 import 路径会变化**：
> ```go
> // v1 版本的导入路径
> import "github.com/some/pkg"
>
> // v2 版本的导入路径（必须加 /v2）
> import "github.com/some/pkg/v2"
> ```
> 这是 Go Module 的设计决定：主版本不兼容，所以路径要区分，避免混用。

Q5: GOPROXY 与国内加速

Go 默认从 `proxy.golang.org` 下载包，在国内无法访问。需要配置代理：

```bash
# 查看当前代理设置
go env GOPROXY

# 设置为国内 goproxy.cn 镜像（推荐）
go env -w GOPROXY=https://goproxy.cn,direct

# 也可以设置 GONOSUMCHECK（跳过某些私有包的校验）
go env -w GONOSUMCHECK=your.private.domain
```

> 📖 **新手友好补充**
>
> 💡 **`direct` 是什么意思？**
> `GOPROXY=https://goproxy.cn,direct` 表示：先从 `goproxy.cn` 下载，如果下载失败（比如包在镜像上找不到），则 `direct` 直接从源码仓库下载。这样私有包也能正常使用。
>
> ⚠️ **新手常见误区**：只设置 `GOPROXY` 而忘记 `GONOSUMCHECK`/`GONOSUMDB`，会导致私有包无法通过 `go.sum` 校验。私有包需要额外配置：
> ```bash
> go env -w GONOSUMDB=your.private.domain
> go env -w GOPRIVATE=your.private.domain
> ```

Q6: 常见工作流总结

```bash
# 1. 新建项目
mkdir myproject && cd myproject
go mod init example.com/myproject

# 2. 写代码，import 需要的包

# 3. 自动拉取依赖
go mod tidy

# 4. 升级某个依赖
go get github.com/gin-gonic/gin@latest

# 5. 降级某个依赖（出问题时回滚）
go get github.com/gin-gonic/gin@v1.8.0

# 6. 清理不再使用的依赖
go mod tidy

# 7. 验证依赖完整性
go mod verify
```

> 📖 **新手友好补充**
>
> ⚠️ **新手常见误区汇总**：
>
> | 误区 | 正确做法 |
> |------|----------|
> | 不提交 `go.sum` 到 Git | **必须提交**，它保证协作者用同一份依赖 |
> | 手动编辑 `go.sum` | 不要手动改，让 Go 自动维护 |
> | 用 `go get` 安装命令行工具 | Go 1.17+ 用 `go install tool@version` |
> | 直接删 `go.mod` 中的 require | 用 `go mod tidy` 自动清理 |
> | 忘记配置 GOPROXY 导致下载失败 | 先设置 `go env -w GOPROXY=https://goproxy.cn,direct` |
